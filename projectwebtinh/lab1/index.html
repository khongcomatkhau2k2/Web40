Các khái niệm cơ bản

  • Repository (repo)
    • Local Repo

      Local Repo là bản sao của repo trên máy tính của bạn. Bạn có thể thực hiện các thay đổi, commit, và kiểm tra lịch sử dự án ngay trên máy tính cá nhân.

    • Remote Repo

      Remote Repo là bản sao của repo trên một máy chủ hoặc dịch vụ trực tuyến như GitHub, GitLab, Bitbucket. Remote repo thường được sử dụng để chia sẻ và phối hợp với các thành viên khác trong nhóm.

  • Commit

    Commit là hành động lưu trữ một bản ghi của các thay đổi đã thực hiện trong repo. Mỗi commit bao gồm một thông điệp mô tả ngắn gọn về thay đổi, giúp theo dõi và hiểu lịch sử của dự án.

  • Clone

    Clone là hành động sao chép một remote repo về máy tính cá nhân, tạo ra một bản sao local repo. Lệnh clone sẽ sao chép toàn bộ lịch sử commit và tất cả các file của dự án.

  • Pull

    Pull là hành động cập nhật local repo với các thay đổi mới nhất từ remote repo. Lệnh pull kết hợp hai lệnh fetch và merge, lấy các thay đổi từ remote repo và hợp nhất chúng với local repo.

  • Push

    Push là hành động gửi các commit từ local repo lên remote repo. Lệnh push giúp cập nhật remote repo với các thay đổi mới nhất từ local repo, cho phép các thành viên khác trong nhóm truy cập và sử dụng những thay đổi này.
